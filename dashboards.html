Dashboards Overview

Dashboards in Evolytix provide an interactive way to visualize, analyze, and manage datasets. Users can configure layouts, add datasets, apply filters, and generate insights using multiple chart types.

1. Creating a New Dashboard

A new dashboard can be initialized by selecting the + New button. This action creates a blank dashboard where users can add and configure visual components.

2. Dashboard Interface Components

Once the dashboard is initialized, users have access to key components:

  • Toolbar: Provides filtering and sorting options for dashboards.
  • Search Bar: Allows users to find specific dashboards.
  • View Options: Switches between different dashboard display formats.
  • Dataset Selector: Displays a list of previously created dashboards.
  • More Options: The three-dot menu provides additional actions.

3. Configuring Dashboard Layout

The dashboard structure is configured using row and panel settings:

  • Row Title: Labels sections of the dashboard.
  • Row Height: Adjusts the height of sections.
  • Panel Width: Controls the width of visualization elements.
  • Item Type: Allows the selection of different chart types.

4. Adding Data to the Dashboard

Datasets can be added by selecting the Add option next to each data source. Users can:

  • Insert dataset tables into the dashboard.
  • Modify dataset parameters.
  • Combine multiple datasets for comparative analysis.

5. Dashboard Data Visualization

Once datasets are placed in the dashboard, users can select from multiple visualization types:

  • Tables: Display raw data in structured columns and rows.
  • Bar Charts: Represent categorical data comparisons visually.
  • Line Charts: Show trends over time.
  • Pie Charts: Illustrate proportional data.
  • Stacked Charts: Compare multiple data categories.

6. Filtering and Sorting Dashboards

Users can apply different filters to their dashboards:

  • Global Filters: Affect all data elements within the dashboard.
  • Panel-Level Filters: Apply only to specific tables or charts.
  • Sorting Options: Sort data within tables and visualizations.

7. Saving and Managing Dashboards

Once a dashboard is configured, users can save it for future use. This process includes:

  • Assigning a Name: Users must provide a descriptive name for easy identification.
  • Saving State: The system stores all visual elements and filters applied to datasets.
  • Sharing Options: Dashboards can be shared with other users for collaboration.

8. Best Practices for Dashboard Design

To ensure dashboards are effective and user-friendly, consider the following best practices:

  • Keep It Simple: Avoid overloading the dashboard with too many elements.
  • Use Consistent Formatting: Ensure uniform color schemes and font sizes for clarity.
  • Prioritize Key Metrics: Focus on displaying the most critical insights first.
